Phase 0 — Project scaffold & CI gates

Goal: compile/run skeleton, enforce quality gates early.
Build:

    Minimal repo layout (/src/{core,http,server,cgi,config,util}, /include, /tests).

    Logger stub (Logger from UML), error pages, Makefile.

    CI: build_cpp98, tests-catch2-cpp11, asan-ubsan, clang-format.

    Interfaces:

        Logger with levels (info/warn/error/debug).

        Result<T> or status enum.

        Error page resolver.

    Tests: CI green on empty tests; clang-format check passes.

Done:

    CI required checks green.

    Repo “Allow auto-merge” on; branch protections set.

Phase 1 — Core loop, poller, connection FSM (State + Strategy)

Goal: accept many clients concurrently without doing HTTP yet.
Build:

    EventLoop, Poller (poll/epoll), TimerWheel, ConnectionManager, unique_fd.

    Connection FSM: READ_HEADERS (stub) → WRITE (echo “hello”) → CLOSE.

    Back-pressure: write buffer watermarks; pause reads when high.

    Timeouts: header read, body read, write flush, idle keep-alive.

    Hooks:

        RateLimiter check stub (no enforcement yet).

        SessionManager placeholder.

    Interfaces: Connection::onReadable/onWritable, EventLoop::register(fd, mask).

Tests:

    Spin server; 1k concurrent connects; verify no leaks (ASan).

Done:

    No blocking syscalls in loop.

    Graceful SIGINT shutdown.

    CPU ~idle when idle.

Phase 2 — HTTP request parsing & response building

Goal: parse HTTP/1.1 correctly with safety caps.
Build:

    HttpParser (request line, headers, CRLF rules).

    Header class: normalized storage via std::unordered_map<std::string, std::string> (case-insensitive keys).

    Caps: max header size, max header count.

    Method enum + validation: GET, POST, DELETE, PUT, PATCH, HEAD.

    Auto “400” / “413” paths.

    HttpResponseBuilder: sets Date, Server, Content-Length (when known).

Interfaces:
HttpRequest (method, target, headers, body_info), HttpResponse.

Tests:

    Fuzz odd headers;

    Oversized header → 431; malformed → 400.

Done:

    Parser is re-entrant, incremental; never throws through the loop.

Phase 3 — Config lexer/parser → ConfigModel (+ validation)

Goal: data-driven server; no hardcoded roots/methods.
Build:

    ConfigLexer → Token; ConfigParser → ConfigModel (servers, locations, CGI config, security).

    VirtualServer and Location classes from UML.

    Support directives:

        roots, index, autoindex

        allowed_methods (incl. PUT/PATCH)

        body limits

        rate_limit (req/sec)

        load_balancing_strategy (round robin / ip_hash)

        session_timeout

        cgi mappings (ext → interpreter path)

    Validation pass: paths exist, no duplicate server_name:port, inheritance rules.

    Error reporting with line/column.

Interfaces:
ConfigModel (read-only view); CgiConfig.

Tests:

    Valid/invalid samples; line pinpoint; inheritance rules.

Done:

    Config errors are human-readable; model immutable after load.

Phase 4 — Routing (Chain of Responsibility) + static files

Goal: route request to Location, serve static files safely.
Build:

    Router:

        Server select by Host+port.

        Location precedence: =, ^~, regex, longest prefix.

    Load balancer (Strategy pattern) for upstream selection if configured.

    StaticFileHandler:

        Canonicalize path, block .. traversal.

        Symlink policy, index file, autoindex.

    Error responses: 403/404/405/301/302; Allow header on 405.

Interfaces:
IHandler (Template Method: prepare/produce), HandlerFactory::create().

Tests:

    Matrix of URIs vs locations; redirects; forbidden traversal; missing file; method not allowed.

Done:

    Paths always resolve under root; precedence matches docs.

Phase 5 — Request body & uploads (Strategy: body readers)

Goal: handle entity bodies safely and predictably.
Build:

    ContentLengthReader and ChunkedReader (reject both headers together).

    Per-location body size limit.

    Spill to temp file when large.

    UploadHandler: supports POST, PUT, PATCH (replace or partial modify).

    Configurable upload dir + permissions.

Interfaces:
IBodyReader strategy selection.

Tests:

    Small/large uploads; limit exceeded → 413; chunked extensions ignored.

Done:

    Back-pressure respected; no event loop stalls.

Phase 6 — CGI (execve) with supervision

Goal: safe, spec-compliant CGI execution.
Build:

    CgiHandler, CgiProcess with pipes.

    Full RFC 3875 env: GATEWAY_INTERFACE, REQUEST_METHOD, etc.

    Timeout + output cap → 504/502; kill & reap.

    Support multiple types: .pl, .py, .php, .sh.

    Parse CGI headers → HttpResponse.

Interfaces:
CgiConfig, ICgiEnvironment.

Tests:

    Good script; missing content-type; infinite loop → timeout; big output cap.

Done:

    No in-process interpreter; zero zombies.

Phase 7 — Sessions & Rate limiting

Goal: manage client state and protect server resources.
Build:

    SessionManager:

        Cookie-based session IDs.

        Expiry from config.

        Per-session storage (in-memory std::unordered_map).

    RateLimiter:

        Token bucket per IP/session.

        Configurable requests/sec.

    Hooked before routing.

Interfaces:
ISessionStore, IRateLimiter.

Tests:

    Session persists across requests.

    Rate limit exceeded → 429.

Done:

    Sessions isolated; DoS attempts mitigated.

Phase 8 — Keep-alive, connection reuse, chunked responses

Goal: production-correct HTTP output.
Build:

    Honor Connection: keep-alive/close.

    Track request count per socket.

    Idle timeout.

    Chunked transfer encoder for unknown length responses.

Interfaces:
ResponseWriter with writeFixed / writeChunked.

Tests:

    Multiple requests on same socket; pipeline-safe; chunked correctness.
